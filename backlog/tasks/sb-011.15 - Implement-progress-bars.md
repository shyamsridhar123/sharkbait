## Description

PRD Section 5.6 requires: "Progress indicators for long operations"

Currently only shows spinner. Need:
- Progress bars for file searches
- Elapsed time display
- Estimated time remaining (if possible)
- Operation-specific progress

## Acceptance Criteria

- [ ] Create `ProgressBar.tsx` component
- [ ] Show progress for file operations
- [ ] Show elapsed time
- [ ] Support determinate (known total) and indeterminate (spinner) modes
- [ ] Cancel button/hint

## Technical Notes

1. Progress bar component:
```tsx
interface ProgressBarProps {
  label: string;
  progress?: number;  // 0-100, undefined = indeterminate
  elapsed?: number;   // milliseconds
  width?: number;     // character width
}

export function ProgressBar({ label, progress, elapsed }: ProgressBarProps) {
  const filled = progress ? Math.floor(progress / 5) : 0;
  const empty = 20 - filled;
  const elapsedStr = elapsed ? `${(elapsed / 1000).toFixed(1)}s` : "";
  
  return (
    <Box>
      <Text>{label} </Text>
      <Text color="green">{"█".repeat(filled)}</Text>
      <Text color="gray">{"░".repeat(empty)}</Text>
      <Text> {progress !== undefined ? `${progress}%` : ""} {elapsedStr}</Text>
    </Box>
  );
}
```

2. Track progress in tools that support it:
- File search: files scanned / total files
- Directory listing: directories traversed

3. For LLM responses: indeterminate (no way to know length)
