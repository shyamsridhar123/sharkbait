## Description

PRD Section 7.3 shows code changes as a diff view:
```
┌─────────────────────────────────────────┐
│ src/auth/login.ts                       │
├─────────────────────────────────────────┤
│ - const token = await authenticate();  │
│ + const token = await authenticate();  │
│ + await saveSession(token);            │
└─────────────────────────────────────────┘
```

Currently, when `edit_file` is called, users don't see what changed.

## Acceptance Criteria

- [ ] Create `DiffView.tsx` component
- [ ] Detect when `edit_file` tool is called
- [ ] Show old vs new content in diff format
- [ ] Use red/green colors for removed/added lines
- [ ] Show file path header
- [ ] Support unified and side-by-side formats

## Technical Notes

1. Capture `oldString` and `newString` from edit_file args
2. Use `diff` library to compute line-by-line changes
3. Render with colors: `-` lines in red, `+` lines in green
4. Show context lines (3 before/after)

## Component API

```tsx
interface DiffViewProps {
  filePath: string;
  oldContent: string;
  newContent: string;
  format?: "unified" | "split";
}
```
