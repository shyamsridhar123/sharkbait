## Description

Support different LLM models for different agent roles.

Per Microsoft Research Magentic-One:
> "Magentic-One is model-agnostic, allowing the integration of heterogeneous models... For the Orchestrator, we recommend a strong reasoning model."

**Model Configuration:**
```typescript
interface ModelConfig {
  orchestrator: ModelSpec;   // Strong reasoning (GPT-4o, Claude Opus)
  primaryAgents: ModelSpec;  // Balanced (GPT-4o, Claude Sonnet)
  parallelModes: ModelSpec;  // Cost-efficient (GPT-4-turbo, Haiku)
}
```

**Model Selection by Task:**
| Task Type | Recommended Model | Reasoning |
|-----------|------------------|-----------|
| Orchestration | Strong (GPT-4o, o1) | Reliable decision-making |
| Complex coding | Full-capability (Codex 5.2) | Deep context understanding |
| Code review (parallel) | Efficient (GPT-4-turbo) | Cost-effective for 3-4 parallel |
| Simple edits | Fast (Haiku-level) | Speed over depth |
| Security analysis | Strong reasoning | Critical accuracy |

**Configuration File:**
```json
// .sharkbait.json
{
  "models": {
    "orchestrator": { "provider": "azure", "deployment": "gpt-codex-5.2" },
    "primaryAgents": { "provider": "azure", "deployment": "gpt-codex-5.2" },
    "parallelModes": { "provider": "azure", "deployment": "gpt-4-turbo" }
  }
}
```

## Acceptance Criteria

- [ ] ModelConfig interface with per-role model specs
- [ ] Model selection based on agent type and task
- [ ] Configuration via `.sharkbait.json` and environment variables
- [ ] Fallback to default model if specific not configured
- [ ] Temperature configuration per task type
- [ ] Cost tracking by model usage
